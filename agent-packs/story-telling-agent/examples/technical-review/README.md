# Technical Review Deck Example

A *design review* deck for a senior-engineering audience. Trades narrative
flair for precision: explicit alternatives, explicit risks, explicit
rollback.

## Files

- `context.md` — paste into `@story-orchestrator`.
- `expected-deck-shape.json` — golden shape (Pyramid framework,
  `technical-slate` design system, 12–18 slides, must include rejected
  alternatives + full risk register + rollback plan).

## What to look for

- Pyramid framework: conclusion first, then supporting evidence, then
  detail.
- No marketing language. The deck-critic should flag any superlative in a
  title.
- Every metric slide carries a source citation (dashboard ID or load-test
  report ID) in the speaker notes.
