# Write + Copy-Back (round-trip) Install Spec (single source) — reference

Overflow reference for `progressive-disclosure/SKILL.md`. Specifies the two
install scripts the **indexer generates** into `<pack-dir>/_install/`:
`install-context-pack.sh` (POSIX) and `install-context-pack.ps1` (PowerShell).
Both implement the **identical** logic below — this reference is the single
source of truth so the two cannot drift. **The indexer generates them; the USER
runs them.** No agent ever copies into a harness dir or runs these scripts.

## Two destinations

1. **Write (generation, done by agents):** the writer + indexer write the pack
   into the **context source-of-truth repo** under
   `context-packs/<slug>-context/`.
2. **Copy-back (round-trip, done by the USER running the script):** the script
   copies `skills/<slug>-context/` from the pack dir into the **code repo's**
   harness skills dir, so the skill is immediately auto-discoverable there.

## Harness-dir resolution order (first match wins)

1. **Explicit override** — a path argument, OR the `CPB_HARNESS_SKILLS_DIR=<path>`
   env var. Use verbatim (create if missing).
2. **Repo-local `.github/skills/`** (preferred) — walk up from the pack dir's
   enclosing repo for `.git/` or `.github/`; target `<repo-root>/.github/skills/`.
3. **User-level `~/.copilot/skills/`** — else, if present or `$HOME` is writable.
4. **Fallback** — if none resolves, **print a message and exit non-zero**, copy
   nothing; the pack stays in the context repo. Re-run with
   `CPB_HARNESS_SKILLS_DIR=<dir>` or a path arg.

The script **prints which rule fired** + the absolute destination.

## Behaviour (idempotent, re-runnable, self-cleaning)

1. Resolve `NS` (namespace) = the pack's `<slug>-context` (read `feature_slug`
   from `../context-pack.json`, else `slugify(basename(pack-dir))`).
2. Resolve `DEST` via the order above; print the rule + destination.
3. Copy `skills/<slug>-context/` → `<DEST>/<slug>-context/`, overwriting in
   place. The source dir is already namespaced (`<slug>-context`), so no rename
   is needed; the destination dir name equals the source.
4. Write a receipt `_install/installed.json` (`feature_slug`, resolved
   `harness_dir`, `installed_at`, copied `skill_dir`, content hash).
5. **Uninstall-on-change**: if a prior receipt recorded a **different**
   `skill_dir` name (the slug was renamed) under the **same** harness dir, delete
   that stale `<DEST>/<old-name>/` — only if it matches the `*-context` shape we
   own. Deleting an absent dir is a no-op.
6. Never delete a dir that does not end in `-context` (never touch another tool's
   skills).

Re-running with an unchanged pack yields an identical destination + receipt — a
no-op in effect.

## `install-context-pack.sh` skeleton (POSIX)

```sh
#!/usr/bin/env sh
# Generated by context-pack-builder. Copies this pack's skill into the resolved
# code-repo harness skills dir, writes a receipt, and removes a stale renamed
# copy (uninstall-on-change). Safe to re-run. The agent generates this file; the
# USER runs it.
set -eu

INSTALL_DIR="$(CDPATH= cd -- "$(dirname -- "$0")" && pwd)"   # the _install/ dir
PACK_DIR="$(dirname -- "$INSTALL_DIR")"
META="$PACK_DIR/context-pack.json"
RECEIPT="$INSTALL_DIR/installed.json"

# 1) Resolve namespace/slug.
SLUG="$(grep -o '"feature_slug"[^,]*' "$META" 2>/dev/null | sed 's/.*: *"//;s/".*//')"
if [ -z "${SLUG:-}" ]; then
  SLUG="$(basename -- "$PACK_DIR" | sed 's/-context$//')"
fi
NAME="$SLUG-context"
SRC="$PACK_DIR/skills/$NAME"
[ -d "$SRC" ] || { echo "ERROR: skill dir not found: $SRC"; exit 2; }

# 2) Resolve destination harness skills dir.
resolve_dest() {
  if [ -n "${1:-}" ]; then echo "explicit-arg $1"; return; fi
  if [ -n "${CPB_HARNESS_SKILLS_DIR:-}" ]; then echo "env-override $CPB_HARNESS_SKILLS_DIR"; return; fi
  d="$PACK_DIR"
  while [ "$d" != "/" ] && [ -n "$d" ]; do
    if [ -d "$d/.git" ] || [ -d "$d/.github" ]; then echo "repo-github $d/.github/skills"; return; fi
    d="$(dirname -- "$d")"
  done
  if [ -d "$HOME/.copilot" ] || [ -w "$HOME" ]; then echo "user-copilot $HOME/.copilot/skills"; return; fi
  echo "none"; return
}
RULE_DEST="$(resolve_dest "${1:-}")"
RULE="${RULE_DEST%% *}"; DEST="${RULE_DEST#* }"
if [ "$RULE" = "none" ]; then
  echo "ERROR: no harness skills dir resolved. Re-run with a path arg or CPB_HARNESS_SKILLS_DIR=<dir>."
  echo "Pack remains in $PACK_DIR awaiting install."
  exit 3
fi
echo "Resolved harness skills dir via rule '$RULE': $DEST"
mkdir -p "$DEST"

# 5) Uninstall-on-change: a previously-installed copy under a DIFFERENT name.
if [ -f "$RECEIPT" ]; then
  prev="$(grep -o '"skill_dir"[^,]*' "$RECEIPT" 2>/dev/null | sed 's/.*: *"//;s/".*//')"
  if [ -n "${prev:-}" ] && [ "$prev" != "$NAME" ]; then
    case "$prev" in
      *-context) if [ -d "$DEST/$prev" ]; then rm -rf "$DEST/$prev"; echo "uninstalled stale: $prev"; fi ;;
    esac
  fi
fi

# 3) Copy the skill into the harness dir (overwrite in place).
rm -rf "$DEST/$NAME"
cp -R "$SRC" "$DEST/$NAME"
echo "installed: $NAME -> $DEST/$NAME"

# 4) Write the receipt.
H="$(ls -lR "$SRC" 2>/dev/null | cksum | cut -d' ' -f1)"
printf '{\n  "feature_slug": "%s",\n  "skill_dir": "%s",\n  "harness_dir": "%s",\n  "installed_at": "%s",\n  "hash": "%s"\n}\n' \
  "$SLUG" "$NAME" "$DEST" "$(date -u +%Y-%m-%dT%H:%M:%SZ)" "$H" > "$RECEIPT"
echo "receipt written: $RECEIPT"
echo "Done."
```

## `install-context-pack.ps1` skeleton (PowerShell)

```powershell
# Generated by context-pack-builder. Copies this pack's skill into the resolved
# code-repo harness skills dir, writes a receipt, and removes a stale renamed
# copy (uninstall-on-change). Safe to re-run. The agent generates this file; the
# USER runs it.
param([string]$Dest)
$ErrorActionPreference = 'Stop'

$InstallDir = Split-Path -Parent $MyInvocation.MyCommand.Path   # the _install/ dir
$PackDir    = Split-Path -Parent $InstallDir
$meta       = Join-Path $PackDir 'context-pack.json'
$receipt    = Join-Path $InstallDir 'installed.json'

# 1) Resolve namespace/slug.
$slug = $null
if (Test-Path $meta) { $slug = (Get-Content $meta -Raw | ConvertFrom-Json).feature_slug }
if (-not $slug) { $slug = (Split-Path -Leaf $PackDir) -replace '-context$','' }
$name = "$slug-context"
$src  = Join-Path $PackDir "skills/$name"
if (-not (Test-Path $src)) { Write-Error "skill dir not found: $src"; exit 2 }

# 2) Resolve destination harness skills dir.
function Resolve-Dest {
  param([string]$Arg)
  if ($Arg)                            { return @('explicit-arg', $Arg) }
  if ($env:CPB_HARNESS_SKILLS_DIR)     { return @('env-override', $env:CPB_HARNESS_SKILLS_DIR) }
  $d = $PackDir
  while ($d -and (Split-Path -Parent $d) -ne $d) {
    if ((Test-Path (Join-Path $d '.git')) -or (Test-Path (Join-Path $d '.github'))) {
      return @('repo-github', (Join-Path $d '.github/skills'))
    }
    $d = Split-Path -Parent $d
  }
  if (Test-Path (Join-Path $HOME '.copilot')) { return @('user-copilot', (Join-Path $HOME '.copilot/skills')) }
  return @('none', '')
}
$r = Resolve-Dest -Arg $Dest
$rule = $r[0]; $target = $r[1]
if ($rule -eq 'none') {
  Write-Error "No harness skills dir resolved. Re-run with -Dest <dir> or `$env:CPB_HARNESS_SKILLS_DIR. Pack remains in $PackDir."
  exit 3
}
Write-Host "Resolved harness skills dir via rule '$rule': $target"
New-Item -ItemType Directory -Force -Path $target | Out-Null

# 5) Uninstall-on-change: previously-installed copy under a DIFFERENT name.
if (Test-Path $receipt) {
  $prev = (Get-Content $receipt -Raw | ConvertFrom-Json).skill_dir
  if ($prev -and $prev -ne $name -and $prev -like '*-context') {
    $old = Join-Path $target $prev
    if (Test-Path $old) { Remove-Item -Recurse -Force $old; Write-Host "uninstalled stale: $prev" }
  }
}

# 3) Copy the skill into the harness dir (overwrite in place).
$dst = Join-Path $target $name
if (Test-Path $dst) { Remove-Item -Recurse -Force $dst }
Copy-Item -Recurse -Force $src $dst
Write-Host "installed: $name -> $dst"

# 4) Write the receipt.
$h = (Get-ChildItem -Recurse $src | Get-FileHash -Algorithm SHA256 | Out-String | Get-FileHash -ErrorAction SilentlyContinue).Hash
[pscustomobject]@{
  feature_slug = $slug
  skill_dir    = $name
  harness_dir  = $target
  installed_at = (Get-Date).ToUniversalTime().ToString('yyyy-MM-ddTHH:mm:ssZ')
  hash         = $h
} | ConvertTo-Json -Depth 5 | Set-Content $receipt
Write-Host "receipt written: $receipt"
Write-Host 'Done.'
```

## Why two native scripts (not one Python script)

Native shells exist on every target with zero install — `sh`/`bash` on
macOS+Linux, PowerShell on Windows. A single `python3` script adds a runtime
dependency not guaranteed on Windows. Maintainers who prefer single-source MAY
ship a `python3` script instead; the eval accepts either form.

## Publishing (independent of copy-back)

Because each pack dir already carries a root `plugin.json` + `skills/`, it is
independently installable as its own Copilot plugin from a marketplace the
developer registers — the copy-back script is only for fast local round-trip
testing.
